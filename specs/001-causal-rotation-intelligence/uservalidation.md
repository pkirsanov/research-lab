# User Validation: 001 Causal Rotation Intelligence

**Evidence source:** [report.md](report.md)  
**Execution plan:** [scopes.md](scopes.md)

## Goal

Validate that the delivered Research Lab capability helps a user discover and challenge an emerging causal rotation without confusing causal evidence with market timing, hiding stale/unavailable inputs, double-counting one event, or rewriting an earlier decision with later facts.

## Journey Steps

1. Open Causal Rotation Lab and use the default Simple view to inspect the highest-ranked eligible candidate or the explicit no-candidate state.
2. Change Discovery/Balanced/Confirmation posture and verify only visibility/market gates change.
3. Open Power view and inspect four clocks, evidence chronology/clusters, regime alternatives, contradictions, unavailable inputs, and falsifiers.
4. Follow the same exposure into Sector Rotation, Global Rotation, or Real Assets and verify the owning market verdict remains independent.
5. Inspect Market Brief coverage and confirm only plan-changing, independently confirming, or falsifying reads consume action/attention space.
6. Freeze a decision, append an outcome/correction, reload, and verify the original record remains unchanged.
7. Exercise catalog/nav/deep links and mobile/desktop layouts on the served production site.

## Checklist

- [x] Planning baseline requires the shared causal foundation and source-time contracts before every UI or consumer overlay.
- [x] Planning baseline covers full owner-tool, Sector/Global/Real Assets, Market Brief, outcome-ledger, registry/docs, and comprehensive validation delivery.
- [x] Planning baseline keeps RRG, country, and real-asset market verdicts owned by their existing tools.
- [x] Planning baseline requires source identity, publication/effective/available times, freshness, classification, contradiction, confirmation, and invalidation for elevated candidates.
- [x] Planning baseline treats missing, stale, unavailable, unverified, and contradictory evidence as visible non-neutral states.
- [x] Planning baseline prevents one catalyst's price/options/ETF reactions from becoming several independent causal reasons.
- [x] Planning baseline includes anti-hindsight regressions that freeze original evidence, thresholds, policy, posture, timing read, and digest.
- [x] Planning baseline requires source-recorded positive observations; malformed/adversarial fixtures prove rejection or demotion only.
- [x] Planning baseline includes exact persistent Regression E2E tests for every scenario and independent RLDATA/RLAPP/registry/Tier-A/owner-verdict canaries.
- [x] Planning baseline requires Simple/Power parity, synchronous nonblank charts, accessibility, no-refetch controls, honest storage failures, and local/Pages desktop/mobile qualification.
