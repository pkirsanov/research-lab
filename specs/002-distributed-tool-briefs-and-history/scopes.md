# Legacy Planning Entry Point

Feature 002 uses the canonical per-scope directory plan at [scopes/_index.md](scopes/_index.md). This compatibility file defines no executable planning inventory.

The canonical plan contains exactly ten executable scope directories. Scope IDs, `scopeDir` values, dependency order, and status must agree across the index, each `scope.md`, `scenario-manifest.json`, and `test-plan.json`.

Registry cardinality is always derived from the current `tools.json` entries and their validated briefing roles. Bond Regime and every other current registry entry participate through that discovery path; no executable scope or test may require a stale literal participant/source count.

The superseded five-scope plan remains available in version control and in `test-plan.json` under `supersededScopes` with `doNotExecute: true`.
