# User Validation

The checklist is checked by default as the planning baseline. A user unchecks an item to report that the delivered behavior does not satisfy the accepted Feature 012 contract.

## Checklist

- [x] Every ordinary registered tool exposes exactly Simple, Power, Brief, and Journey through one shared control; Market Action Center exposes exactly Brief, Portfolio, Red Alert, and Journey.
- [x] Every ordinary Simple experience is a distinct steerable owner model or seeded simulation with meaningful controls, baseline/current sensitivity, provenance, uncertainty, and limitations.
- [x] Every Power analytical item explains its exact current value by pointer, keyboard, and touch, with an adjacent same-data alternative for charts.
- [x] Every current authored Brief uses one validated owner read plus one frozen qualified WebEvidenceBundle, cites material claims, and gives the author no web, shell, write, model, key, or private-state authority.
- [x] Every ordinary tool offers at least two concrete goals and the shared Journey runtime resumes, backtracks, invalidates dependent conclusions, exports typed packets, and never executes.
- [x] `market-brief.html` remains usable under the visible Market Action Center name and preserves its registry identity, four ET windows, action gates, bookmarks, history, and deep links.
- [x] Public watchlist rows never imply holdings; private workspace rows remain local, use Feature 008 as the only store, and state that no scheduled personalized LLM Brief exists.
- [x] Red Alerts are discovered dynamically, require independent citations and observable market evidence, remain falsifiable/non-alarmist, and may truthfully be empty.
- [x] Global Journeys prepare the next session, triage current actions, investigate latent risk, and stress private scope locally without mutation or execution.
- [x] Keyed providers, Yahoo, daily snapshots, options, owner reads, publications, private state, and QF remain under their existing owners with exact dependency gates and no bypass.
- [x] Desktop, narrow mobile, 200% zoom, keyboard, touch, reduced motion, focus return, ticker links, disclosures, and canvas fallbacks preserve all required content without overlap or body-level horizontal scrolling.
- [x] Research Lab remains static and build-free; QF portability is contract-semantic only and copies no source in either direction.
